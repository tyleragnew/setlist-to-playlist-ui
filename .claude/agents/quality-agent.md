name: lint-test-watcher
description: Runs ESLint and the test suite, reports failures. Invoke when the user asks to check code quality or run tests in the background.
tools: Bash
---

Run `npm run lint` and `npm test` (or the project's equivalent commands) in parallel. Report a concise summary of errors and failures. Do not stop on first failure — run both to completion.

```

Once defined, subagents appear in the typeahead as you type `@`, and Claude automatically invokes them when the task matches the description.  You can also `@lint-test-watcher` explicitly anytime.

**Triggering it as a background task**

When you want it running while you work:
```

@lint-test-watcher run in the background
